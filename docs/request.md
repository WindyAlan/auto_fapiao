我有一个发票自动重命名和校验的需求：

# 任务1 根据合同号和ocr识别结果自动获取甲方、乙方合同号，对源文件重命名。
首先我有一个文件夹，其中包含多个pdf文件，文件名有两种形式：
1. 2100002601080A_xxx.pdf，其中的2100002601080A是乙方合同号。有一个对应的excel表格，根据乙方合同号，可以在合同名称这一栏获取甲方的合同号。甲方合同号有两种形式：
	1. YW-7871Y00200172-0C1911-651400P1，提取其中的7871Y00200172
	2. W-SZYG4502802-4-S292912F-3000P，提取其中的SZYG4502802-4
抽取完成后，重命名源文件为{甲方合同号}-2100002601080A_xxx.pdf

2. dzfp_26317000001473818238_xxx.pdf，这种文件乙方合同号不在文件名中，需要ocr识别发票的备注这一栏获取乙方合同号，再去索引甲方合同号，然后重命名文件为{甲方合同号}-{乙方合同号}_26317000001473818238_xxx.pdf。

# 任务2 根据表格和发票的ocr识别结果，验证表格中填写的发票信息是否正确，错误的话进行改正，然后告知使用者进行二次排查。
我有一个excel表格，其中的AF、BZ、CA、CB、CC、CD（第四行起才是真实数据），分别对应甲方合同号，开票数量、发票号、发票日期、税金金额、含税金额。根据任务1识别到的甲方合同号查到其他信息，然后和实际发票pdf文件ocr识别的结果进行比对校验，出现问题尝试自动修复excel内容，但同时也要告诉用户修复了哪些内容。OCR识别置信度很低时，不自动修复，提示用户自己修复。
