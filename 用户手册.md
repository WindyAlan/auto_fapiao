# 发票自动工具 - 用户手册

## 第一步：安装（只需做一次）

1. 确认电脑上已经安装了 Python 3.12 和 uv（找技术人员帮忙装好）
2. 把整个 `auto_fapiao` 文件夹复制到你的电脑上
3. 双击文件夹里的 `run.bat`，等一会儿看到窗口弹出来就说明装好了

## 第二步：重命名发票

1. **双击 `run.bat`**，会弹出一个窗口
2. 在 **Step 1** 区域：
   - 点 **Browse** 选你的发票 PDF 文件夹
   - 点 **Browse** 选合同号索引 Excel 文件（就是那个有合同名称的表）
3. 点 **Start Rename**
4. 等下面日志区域显示 "Rename complete!" 就完成了
5. 去你的发票文件夹旁边，会多出一个 `原文件夹名_Renamed` 文件夹，里面就是重命名好的文件

**举个例子：**
- 你的发票在 `D:\ invoices\ pdfs\` 文件夹里
- 重命名后会生成 `D:\ invoices\ pdfs_Renamed\` 文件夹
- 里面会有 `rename_report.txt`，告诉你哪些成功了、哪些失败了

## 第三步：校验发票信息

1. 在 **Step 2** 区域：
   - 点 **Browse** 选**重命名后的** PDF 文件夹（就是上一步生成的 `_Renamed` 文件夹）
   - 点 **Browse** 选发票验证 Excel 文件
2. 点 **Start Verify**
3. 等下面日志区域显示 "Verify complete!" 就完成了
4. 去发票验证 Excel 所在的文件夹，会多出一个 `verify_report.txt` 文件

**report 文件里会告诉你：**
- 哪些发票信息完全正确
- 哪些有差异，Excel 填的和 PDF 上实际印的不一样（需要你自己去核实修改）
- 哪些文件无法识别

## 常见问题

**Q: 双击 run.bat 没反应？**
A: 右键点击 run.bat → 选择"以管理员身份运行"，或者找技术人员检查 Python 是否装好。

**Q: 点了 Start 之后很久没反应？**
A: 正常的，第一张发票需要加载 OCR 模型，大概等 10-30 秒。后面就快了。

**Q: 报告里很多"需手动核查"怎么办？**
A: 打开 report 文件，找到"需手动核查"的部分，对照 PDF 发票原件，手动修改 Excel 里的数据即可。

**Q: 我只想重命名不想校验，可以吗？**
A: 可以，只做 Step 1 就行，Step 2 不用管。

**Q: 原始的 Excel 文件会被修改吗？**
A: 不会。工具不会修改你的任何原始文件。重命名是复制到新文件夹，校验结果只写在 report 文本文件里。
